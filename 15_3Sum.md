# 15. 3Sum

**LeetCode:** https://leetcode.com/problems/3sum/

**Difficulty:** Medium

---

# Pattern(s)

- Sorting
- Two Pointers
- Array

---

# Pattern Recognition

Look for this approach when:

- The problem asks for triplets.
- The sum of three elements equals a target (usually 0).
- Duplicate triplets should be avoided.
- Sorting can simplify searching.

---

# Intuition

We need to find all unique triplets whose sum is `0`.

The brute force approach checks every possible triplet.

A better approach fixes one element and searches for the remaining two using a HashSet.

The optimal approach first sorts the array.

Then,

- Fix one element.
- Use two pointers to find the other two elements.
- Skip duplicates to avoid repeated triplets.

Sorting makes duplicate handling and searching much easier.

---

# Brute Force

## Idea

Generate every possible triplet.

If the sum is `0`, sort the triplet and store it in a Set to avoid duplicates.

## Algorithm

1. Generate all triplets.
2. Check whether their sum is `0`.
3. Sort the triplet.
4. Insert into a Set.
5. Return all unique triplets.

### Java Code

```java
class Solution {
    public List<List<Integer>> threeSum(int[] nums) {

        Set<List<Integer>> set = new HashSet<>();

        int n = nums.length;

        for(int i = 0; i < n; i++) {

            for(int j = i + 1; j < n; j++) {

                for(int k = j + 1; k < n; k++) {

                    if(nums[i] + nums[j] + nums[k] == 0) {

                        List<Integer> temp = Arrays.asList(nums[i], nums[j], nums[k]);

                        Collections.sort(temp);

                        set.add(temp);

                    }

                }

            }

        }

        return new ArrayList<>(set);
    }
}
```

### Complexity Analysis

**Time Complexity:** **O(n³)**

- Three nested loops generate all triplets.

**Space Complexity:** **O(n)** *(excluding output)*

- HashSet stores unique triplets.

# Better Approach

## Idea

Fix one element.

Now the problem becomes:

> Find two numbers whose sum equals `-nums[i]`.

Use a HashSet while traversing the remaining array.

Store every visited number.

For every new number, check whether its complement already exists.

Again, sort the triplet before inserting into a Set to remove duplicates.

## Algorithm

1. Fix one element.
2. Create a HashSet.
3. Traverse the remaining array.
4. Compute the required complement.
5. If found, store the triplet.
6. Otherwise, insert the current element into the HashSet.

### Java Code

```java
class Solution {
    public List<List<Integer>> threeSum(int[] nums) {

        Set<List<Integer>> ans = new HashSet<>();

        int n = nums.length;

        for(int i = 0; i < n; i++) {

            HashSet<Integer> set = new HashSet<>();

            for(int j = i + 1; j < n; j++) {

                int third = -(nums[i] + nums[j]);

                if(set.contains(third)) {

                    List<Integer> temp =
                            Arrays.asList(nums[i], nums[j], third);

                    Collections.sort(temp);

                    ans.add(temp);

                }

                set.add(nums[j]);

            }

        }

        return new ArrayList<>(ans);
    }
}
```

### Complexity Analysis

**Time Complexity:** **O(n² log 3)**

- Outer loop: **O(n)**
- Inner loop: **O(n)**
- Sorting a triplet is constant time since there are only 3 elements.

Overall,

```
O(n²)
```

**Space Complexity:** **O(n)**

- HashSet stores previously seen elements.

# Optimal Approach

## Idea

Sort the array first.

Now fix one element.

Use two pointers:

- `left`
- `right`

Calculate

```
sum = nums[i] + nums[left] + nums[right]
```

- If sum is too small, move `left`.
- If sum is too large, move `right`.
- If sum is `0`, store the triplet and skip duplicates.

Sorting allows duplicate triplets to be skipped easily.

## Algorithm

1. Sort the array.
2. Traverse every index as the first element.
3. Skip duplicate first elements.
4. Place two pointers.
5. Calculate the sum.
6. Move pointers according to the sum.
7. Skip duplicate second and third elements.
8. Return the answer.

### Java Code

```java
class Solution {
    public List<List<Integer>> threeSum(int[] nums) {

        Arrays.sort(nums);

        List<List<Integer>> ans = new ArrayList<>();

        int n = nums.length;

        for(int i = 0; i < n - 2; i++) {

            if(i > 0 && nums[i] == nums[i - 1]) {
                continue;
            }

            int left = i + 1;
            int right = n - 1;

            while(left < right) {

                int sum = nums[i] + nums[left] + nums[right];

                if(sum < 0) {

                    left++;

                }
                else if(sum > 0) {

                    right--;

                }
                else {

                    ans.add(Arrays.asList(
                            nums[i],
                            nums[left],
                            nums[right]
                    ));

                    left++;
                    right--;

                    while(left < right &&
                          nums[left] == nums[left - 1]) {
                        left++;
                    }

                    while(left < right &&
                          nums[right] == nums[right + 1]) {
                        right--;
                    }

                }

            }

        }

        return ans;
    }
}
```

### Complexity Analysis

**Time Complexity:** **O(n²)**

- Sorting: **O(n log n)**
- Two-pointer search for every element: **O(n²)**

Overall,

```
O(n²)
```

**Space Complexity:** **O(1)** *(excluding output)*

- No extra data structure is used.
